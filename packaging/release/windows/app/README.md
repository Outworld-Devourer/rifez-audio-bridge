# Windows App Release Payload

This folder is intended to contain the Release payload for the Windows tray companion app.

## Intended contents

The Windows app payload should include all files required to run the tray app outside the development environment.

Typical contents include:

- main tray app executable
- dependent DLLs
- configuration/runtime support files generated by the Release build
- any additional assets that are not already embedded

## Current notes

The tray app currently uses embedded icon resources, which reduces asset path dependency issues between Debug and Release builds.

## Validation

Before staging files here, verify:

- Release build succeeds
- tray app launches from the Release output folder
- tray icons appear correctly
- status window opens correctly
- reconnect and auto-restore behavior work
- exit fully closes the process
